# Skills 说明

本文档用于说明当前仓库内两个核心 skill 的定位、适用场景和使用方式：

- `generate-prd-mermaid`
- `mermaid-diagrams`

这两个 skill 的关系是：

- `generate-prd-mermaid` 负责把模糊业务想法整理成结构化需求与产品设计文档。
- `mermaid-diagrams` 负责生成和约束 Mermaid 图表语法。
- 前者在需要出图时会调用后者的规则，因此两者是“文档生成 + 图表生成”的配套关系。

---

## 1. generate-prd-mermaid

### Skill 名称

- `generate-prd-mermaid`

### 核心作用

把一句话需求、模糊项目想法或半成型业务描述，整理成可评审的需求与产品设计初稿，并自动补充 Mermaid 图表。

### 适用场景

- 需要快速生成 PRD、需求文档、立项草案、产品方案。
- 需要把业务描述拆成模块、流程、状态、页面结构。
- 需要在文档中同步产出 Mermaid 流程图、结构图或时序图。
- 用户输入不完整，但仍希望先拿到一版可讨论的初稿。

### 主要特点

- 支持从模糊输入生成结构化输出。
- 输入不完整时，不会静默补全，而是先输出 `输入缺口提醒`。
- 明确区分 `问题`、`目标`、`范围` 和 `方案`，避免“把方案当需求”。
- 默认输出中文文档，并可附带 1 到 3 个 Mermaid 图。
- 支持 `MVP`、`本期范围`、`非本期范围`、`待确认项` 等产品文档常用结构。

### 推荐输入

建议尽量提供以下信息：

- 项目名称
- 项目类型
- 业务背景
- 目标用户
- 核心问题
- 希望实现的功能
- 成功指标
- 上线约束
- 已有系统或现有流程

### 输出内容

通常会输出以下部分：

- 输入缺口提醒
- 需求摘要
- 项目背景与问题定义
- 用户与角色分析
- 业务目标与成功指标
- 范围定义与 MVP 建议
- 功能模块拆解
- 页面结构或信息架构
- 业务规则、异常场景、风险依赖
- Mermaid 图表
- 信息缺口与待确认项

### 适合的用户表达方式

例如：

```text
使用 $generate-prd-mermaid，帮我把一个仓库库存管理系统整理成需求文档和产品设计，并补 Mermaid 图。
```

```text
使用 $generate-prd-mermaid。
我想做一个内部审批系统，先帮我生成一版 PRD 和流程图。
```

### 适用价值

- 适合做立项前方案收敛。
- 适合做产品讨论、研发评审、汇报材料前置文档。
- 适合在信息不全时先输出一版骨架，提升讨论效率。

## 2. mermaid-diagrams

### Skill 名称

- `mermaid-diagrams`

### 核心作用

专门用于生成、规范和优化 Mermaid 图表，适合输出流程图、时序图、类图、ER 图、C4 架构图等基于文本的结构化图表。

### 适用场景

- 需要把系统流程、业务交互、数据结构或架构关系画成 Mermaid 图。
- 需要为 PRD、技术方案、README、系统设计文档补图。
- 需要让 Mermaid 图语法更稳定、结构更清晰、图型选择更准确。

### 支持的主要图类型

- `flowchart`
- `sequenceDiagram`
- `classDiagram`
- `erDiagram`
- `C4Context / C4Container / C4Component`
- `stateDiagram`
- 其他 Mermaid 支持的标准图型

### 主要特点

- 按图类型拆分了参考规则文件，便于按需加载。
- 提供基础语法、结构规则、示例和最佳实践。
- 包含高级主题、样式、布局、themeVariables 等扩展说明。
- 适合单独出图，也适合作为其他 skill 的图表规则依赖。

### 推荐输入

尽量说明以下内容：

- 想画什么图
- 图的用途
- 核心参与者、节点、实体或模块
- 流程顺序、分支条件或关系类型
- 是否需要保持简单、可渲染优先

### 输出内容

通常输出为：

- Mermaid 代码块
- 图型选择说明
- 节点或参与者说明
- 必要时附带简短解释

### 适合的用户表达方式

例如：

```text
使用 $mermaid-diagrams，帮我画一个用户下单到支付完成的时序图。
```

```text
使用 $mermaid-diagrams，帮我把库存系统的模块结构整理成 flowchart。
```

### 适用价值

- 适合把抽象说明转成结构化图表。
- 适合技术文档、架构文档、产品文档统一图表表达。
- 适合与 `generate-prd-mermaid` 配合使用，形成“先出文档，再出图表”的完整链路。

---

## 两个 Skill 如何配合

推荐的使用方式：

1. 先使用 `generate-prd-mermaid` 生成需求与产品设计初稿。
2. 在该过程中由 `generate-prd-mermaid` 按需读取 `mermaid-diagrams` 的图表规则。
3. 如果后续只想单独优化某一张图，再使用 `mermaid-diagrams` 单独细化。

### 简单理解

- 想要“完整需求方案 + 图表”：用 `generate-prd-mermaid`
- 想要“只画 Mermaid 图”：用 `mermaid-diagrams`

---

## 建议

- 如果输入业务信息较少，优先用 `generate-prd-mermaid`，先拿一版带假设的初稿。
- 如果你已经有完整文档，只差图表表达，直接用 `mermaid-diagrams` 更合适。
- 对复杂需求，不要试图用一张图塞下全部内容，应拆成主流程图、结构图、时序图等多张图。
