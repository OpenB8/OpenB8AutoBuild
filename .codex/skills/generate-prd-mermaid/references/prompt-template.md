# Prompt 模板

## 通用模板

```text
使用 $generate-prd-mermaid，根据以下输入生成可评审的需求与产品设计初稿，并自动补 Mermaid 图表。

输入信息：
项目名称：
项目类型：
业务背景：
目标用户：
核心问题：
希望实现的功能：
成功指标：
上线约束：
已有系统/流程：
补充说明：

输出要求：
1. 如果输入不完整，先输出“输入缺口提醒”，说明缺失字段、影响和默认处理。
2. 若信息缺口不阻塞第一版方案，继续生成需求摘要和完整正文；若阻塞核心判断，先提出最少必要问题。
3. 明确区分问题定义、业务目标、功能范围和解决方案。
4. 输出需求与立项、产品设计、Mermaid 图表、信息缺口四部分。
5. 至少生成 1 个 Mermaid 图；复杂需求优先给流程图 + 状态图或时序图。
6. Mermaid 图放在独立代码块中，使用兼容性高的基础语法。
7. 对已有系统迭代，标出新增、修改、保持不变。
```

## 极简输入模板

```text
使用 $generate-prd-mermaid。
这是一个关于【业务主题】的想法：
【一句话描述】

如果信息不足，先提醒缺什么，再补出合理假设，然后生成：
- 需求摘要
- MVP 范围
- 产品设计骨架
- Mermaid 主流程图
- 待确认项
```
